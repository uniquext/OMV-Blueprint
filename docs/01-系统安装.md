## 2. 首次启动：基础设施固化

安装完成后首次进入 OMV Web UI，必须**优先**完成网络固化与时间同步，确保后续的一系列自动化脚本及插件安装具备稳定的基础运行环境。

### 2.1 网络静态化 (IP Static)

在 **网络** → **接口** 中修改主网卡配置。将网卡从 DHCP 切换为 Static 模式，防止因 DHCP 续租带来的 IP 漂移导致 SSH 连接及 Web UI 中断。

| 配置项 | 推荐值 | 修改建议 | 说明 |
| :--- | :--- | :---: | :--- |
| **IPv4 模式** | `静态/Static` | 🔴 **必改** | OMV 后台依赖稳定的宿主机 IP |
| **DNS 服务器** | `223.5.5.5` | 🟢 建议 | 推荐使用阿里公共 DNS，国内解析响应极快 |

> [!IMPORTANT]
> **连线风险提示**：若不设置静态 IP，后续在通过 SSH 执行长耗时任务（如 Docker 镜像拉取或内核更新）时，一旦 DHCP 租期满触发 IP 强制刷新，当前的 SSH 会话将瞬间断开且无法原位恢复。

### 2.2 时区与 NTP 同步

进入 **系统** → **工作台** → **日期和时间**，校准系统时钟。**时间偏移是导致 APT 签名校验失败、GPG 报错及容器鉴权失效的头号隐形杀手。**

| 配置项 | 推荐值 | 修改建议 | 说明 |
| :--- | :--- | :---: | :--- |
| **时区** | `Asia/Shanghai` | 🟢 保持默认 | 确保系统日志与现实时间对齐 |
| **NTP 服务器** | `ntp.ntsc.ac.cn` | 🔴 **必填** | 中国科学院国家授时中心（国内访问极快） |

> [!TIP]
> 完成上述两项底层设置并**应用更改**后，建议执行一次 `reboot` 重启系统，验证静态 IP 是否生效以及系统时间能否在掉电重启后成功自动对齐。

## 3. 笔记本休眠关闭（按需）

> [!NOTE]
> 仅适用于使用笔记本电脑充当服务器的场景。台式机或工控机用户请跳过本节。

服务器必须保持 7×24 小时运行，笔记本默认的合盖休眠行为必须在初期直接干掉。

```bash
# 编辑 logind 配置文件
sudo vim /etc/systemd/logind.conf

# 找到并修改以下两行（去掉前面的 # 注释符）
HandleLidSwitch=ignore
LidSwitchIgnoreInhibited=yes

# 重启服务使配置生效
sudo systemctl restart systemd-logind.service
```

| 参数 | 值 | 含义 |
| :--- | :--- | :--- |
| `HandleLidSwitch` | `ignore` | 合上笔记本盖时不执行任何操作 |
| `LidSwitchIgnoreInhibited` | `yes` | 即使有抑制锁，也优先执行 `ignore` 策略 |
