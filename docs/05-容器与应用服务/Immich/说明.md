# 05.2 — Immich (高性能自托管照片与视频管理)

Immich 是一个高性能的自托管照片和视频管理解决方案，旨在提供类似于 Google Photos 的体验，具备极佳的响应速度和全功能的移动应用。

## 1. 部署架构 (Deployment)

本项目适配了 OMV 容器部署规范，对数据进行了三层逻辑分离，严禁硬编码 UUID 路径：

- **服务自产层 (Service Local)**：存放在 Compose 目录下的相对路径。
    - `./data`: PostgreSQL 数据库文件。
    - `./model-cache`: 机器学习模型缓存（避免重复下载）。
- **公共资源层 (AppData)**：引用 `${COMPOSE_DATA_PATH}`。
    - *本项目暂无共享脚本需求。*
- **媒体资源层 (Resource)**：存储真实照片与视频文件。
    - `${UPLOAD_LOCATION}`: 映射至 `${PHOTO_PATH}` (由 `global.env` 定义，通常指向 `Resource/DCIM`)。

## 2. 快速启动 (Quick Start)

1. **配置文件**：主要入口为 [Immich.yml](../../../Compose/Immich/Immich.yml)。
2. **凭据准备**：
   - [!IMPORTANT]
   - 必须确保 `secrets/private.env` 中包含 `SECRET_IMMICH_DB_PASSWORD` 的定义。
   - 必须确保 `global.env` 中的 `PHOTO_PATH` 指向正确的存储盘。
3. **部署指令**：
   在 OMV Compose 界面拉起该堆栈。由于定义了 `networks: default`，堆栈会自动创建 `immich_network` 桥接网络。

## 3. 详细配置参数表 (Configuration Table)

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| 容器命名 | `${SERVER_HOSTNAME}` | 🟢 | 建议保持 `immich-server` |
| 主机端口 | `2283` | 🟡 | 若冲突可修改，容器内固定为 2283 |
| 上传路径 | `${UPLOAD_LOCATION}` | 🔴 | 默认指向全局 DCIM，如有特殊需求可在 `Immich.env` 覆盖 |
| 数据库密码 | `${DB_PASSWORD}` | 🔴 | 必须在 `private.env` 中安全定义 |
| 镜像版本 | `${IMMICH_VERSION}` | 🟡 | 当前固定为 `v2.5.6` 以保证稳定性 |
| 信任代理 | `${IMMICH_TRUSTED_PROXIES}` | 🟡 | 填入 Cloudflare 隧道或 Nginx 反代网段 |

## 4. 外部协作与优化 (Optimization)

### 网络与 DNS
- **内部解析**：堆栈内服务通过 `DB_HOSTNAME` 等变量互相访问，无需依赖宿主机 IP。

### 数据库完整性
- **防损坏**：配置了 `POSTGRES_INITDB_ARGS: "--data-checksums"`，在存储层探测位翻转，确保照片元数据安全。

## 5. 运维与灾备 (Maintenance)

### 数据库恢复
如果需要进行系统重装或环境迁移，**仅创建同名用户无法恢复相册与人脸数据**。必须使用备份目录下的 `.sql.gz` 文件进行恢复。
