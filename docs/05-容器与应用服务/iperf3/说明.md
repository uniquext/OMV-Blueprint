# iperf3 - 网络带宽测试工具

> iperf3 是业界标准的网络带宽测试工具，用于测量 TCP/UDP 的最大吞吐量，支持多线程、双向测试和实时报告。
>
> - **官方文档**：[iperf3 用户手册](https://software.es.net/iperf/)
> - **项目地址**：[esnet/iperf](https://github.com/esnet/iperf)

---

## 一、 部署架构 (Deployment)

iperf3 采用单容器 Host 网络模式部署，绕过 Docker NAT 开销以获得准确的带宽测试结果。

### 1.1 容器架构说明

- **iperf3**：以服务端模式 (`-s`) 持续运行，监听 5201 端口等待客户端连接进行带宽测试

### 1.2 路径映射逻辑

- **Service Local (服务自产层)**：无持久化数据需求，iperf3 为无状态测试工具
- **AppData (公共资源层)**：无
- **Resource (媒体资源层)**：无

## 二、 快速启动 (Quick Start)

> [!IMPORTANT]
> **前置检查**：部署前，请确保 `global.env` 中相关变量已定义并正确指向目标物理路径。

1. 查阅与验证配置逻辑：
   - 配置文件：[Iperf3.yml](../../../Compose/Iperf3/Iperf3.yml)
2. 在 OMV Web UI 的 Compose 标签页，加载此 `Iperf3.yml` 文件。
3. 点击 `Up`（启动）即可。服务端将在 `5201` 端口监听。
4. 从客户端发起测试：
   ```bash
   # 基础 TCP 带宽测试
   iperf3 -c <OMV_IP>

   # UDP 带宽测试
   iperf3 -c <OMV_IP> -u -b 1G

   # 双向测试（同时测上下行）
   iperf3 -c <OMV_IP> -R
   ```

## 三、 详细配置参数表 (Configuration Table)

所有环境变量声明于 `iperf3.yml`，且严格依赖宿主机的 `global.env` 透传。

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| **基础配置** | | | |
| 容器映像 | `networkstatic/iperf3:latest` | 🟢 保持默认 | 轻量级 iperf3 镜像 |
| 端口监听 | `5201` | 🟡 按需调整 | Host 模式直通，iperf3 默认端口 |
| 外部网络模式 | `host` | 🟢 保持默认 | 绕过 Docker NAT，确保测试精度 |
| **环境变量 (Environment)** | | | |
| 权限 UID | `PUID` | 🟢 保持默认 | 指向 `${PUID}`（OMV `appuser` 的数值 1001） |
| 权限 GID | `PGID` | 🟢 保持默认 | 指向 `${PGID}`（`users` 组的数值 100） |
| 时区同步 | `TZ` | 🟢 保持默认 | 指向 `${TZ}`（如 `Asia/Shanghai`），防止时间戳错位 |
| **目录挂载 (Volumes)** | | | |
| 主机时间挂载 | `${LOCALTIME_PATH}` | 🟢 保持默认 | `/etc/localtime` 确保日志的时间属性严格一致 |

## 四、 首次登录配置 (Initial Setup)

iperf3 为命令行工具，无 Web 界面，无需登录配置。部署后即可直接从客户端发起带宽测试。

---

## 五、 日常运维

- **启动服务端**：在 OMV Compose 中 `Up` 启动容器
- **停止服务端**：在 OMV Compose 中 `Down` 停止容器，不测试时可关闭以释放资源
- **查看实时连接**：`docker logs iperf3` 查看客户端连接与测试结果日志
- **修改监听端口**：编辑 `Iperf3.yml` 中 `command` 为 `["-s", "-p", "端口号"]`
