# Servarr - 媒体自动化大一统

> Servarr 是对 `*arr` 系列媒体获取软件（Sonarr, Radarr, Prowlarr 等）的统称。由于它们高度互赖、底层同源（NzbDrone），本项目采用将其合并为单一 `Servarr.yml` 的方式部署，由 OMV 统一管理和启停。实现从：媒体搜索 → 索引抓取 → 自动下载 → 属性刮削 → 字幕匹配 → 质量同步 的全流程闭环。
>
> - **项目地址**：[Servarr Wiki](https://wiki.servarr.com/)
> - **TRaSH Guide**：[recyclarr.dev](https://recyclarr.dev/guide/)

---

## 一、 部署架构 (Deployment)

Servarr 采用 **9 容器 Bridge 模式** 部署，统一运行在 `servarr-network` 隔离网络内，各容器通过内部 `hostname` 进行无缝通信。

### 1.1 容器架构说明

**基础支撑层**：
- **flaresolverr**：爬虫验证码突破代理，绕过 Cloudflare 等站点防火墙
- **qbittorrent**：BT 下载客户端，提供 Web UI 管理界面

**核心管理层**：
- **prowlarr**：全局索引器同步与管理，统一分发索引器到各 Arr 应用
- **sonarr**：剧集 (TV Shows) 自动化管理，自动追剧与下载
- **radarr**：电影 (Movies) 自动化管理，自动获取与刮削
- **whisparr**：成人媒体自动化管理

**后处理与辅助层**：
- **bazarr**：字幕自动化搜寻与精准匹配，监控下载任务自动下载字幕
- **jellyseerr**：跨平台媒体订阅与请求发现，为 Jellyfin 用户提供 Netflix 式体验

**工具/同步层**：
- **recyclarr**：无 Web UI，基于 CLI 定时同步 TRaSH Guide 最佳质量实践配置
- **qbitrr**：磁力链接与失速管理，自动清理无速度种子并触发换种

### 1.2 路径映射逻辑

- **Service Local (服务自产层)**：
  - `./config/[服务名]` 目录：各服务的运行时配置与 SQLite 数据库
- **Resource (媒体资源层)**：
  - `${RESOURCE}/Media`：海量媒体库物理存放路径，供 Sonarr/Radarr/Bazarr 访问
  - `${RESOURCE}/Downloads`：临时与持久化下载目录，供 qBittorrent 写入、Sonarr/Radarr 读取

## 二、 快速启动 (Quick Start)

> [!IMPORTANT]
> **前置检查**：部署前，请确保 `global.env` 中 `RESOURCE`、`PUID`、`PGID`、`TZ` 等变量已定义并正确指向目标物理路径。同时确保 `private.env` 中已设置 `SERVARR_TARGET_PASS` 和 `SERVARR_QB_PASS` 等。

1. 查阅与验证配置逻辑：
   - 配置文件：[Servarr.yml](../../../Compose/Servarr/Servarr.yml)
   - 环境变量：[Servarr.env](../../../Compose/Servarr/Servarr.env)
2. 在 OMV Web UI 的 **Services > Compose > Files** 中，加载此 `Servarr.yml` 文件。
3. 点击 `Up`（启动）堆栈，等待容器运行 **约 1 分钟**（让容器生成底层 SQLite 数据库和配置文件）。
4. 进入命令行，在 `Compose/Servarr/` 目录下执行一键整合脚本：

    ```bash
    ./startup.sh
    ```

**✅ 脚本将自动完成以下事情**：

一键整合脚本按 6 个阶段依序执行，覆盖从凭据初始化到应用融合的全链路配置：

- [✓] **初始化与认证**：提取所有服务 API Key，重置 qBittorrent 临时密码，为 Sonarr / Radarr / Whisparr 注入表单认证。
- [✓] **索引器与整合**：在 Prowlarr 中注册 FlareSolverr 代理、qBittorrent 下载器，绑定各 Arr 应用为接收端，并注入常用公共索引器。
- [✓] **基础配置注入**：为 Sonarr / Radarr / Whisparr 统一配置中文 UI、命名规范、默认根目录、下载客户端及首选 Torrent 15分钟延迟协议。
- [✓] **质量同步与 CF 导入**：生成 Recyclarr 配置并执行首次 TRaSH Guide 质量同步，导入中文字幕 Custom Formats。
- [✓] **字幕系统配置**：为 Bazarr 创建语言配置文件（简繁中 + 英文）、启用默认字幕源、绑定 Sonarr / Radarr 连接。
- [✓] **Jellyseerr 多平台融合**：完成 Jellyfin 登录与媒体库同步、Radarr / Sonarr 接入及最终初始化。

**⚠️ 需手动完成的事项**：
- [ ] **根目录调整**：脚本默认设置 `/media` 为统一根目录，请根据实际媒体库结构前往各 Arr 应用调整（参考 [4.1 根目录调整](#41-根目录调整)）
- [ ] **索引器配置**：前往 Prowlarr Web UI 添加私有索引器（参考 [4.2 Prowlarr 索引器](#42-prowlarr-索引器)）
- [ ] **字幕源补充**：前往 Bazarr 添加 OpenSubtitles 等缺失字幕源（参考 [4.3 Bazarr 字幕源配置](#43-bazarr-字幕源配置)）

## 四、 首次登录配置 (Initial Setup)

### 4.1 根目录调整

脚本默认为 Sonarr / Radarr / Whisparr 设置 `/media` 作为统一根目录。在实际生产环境中，您通常需要根据媒体类型拆分为多个子目录。

访问各 Arr 应用的 `Settings → Media Management → Root Folders`，按需添加或修改，例如：
- `/media/Anime/TV番剧`
- `/media/Anime/剧场版电影`
- `/media/TV`
- `/media/Movie`

> [!WARNING]
> **写入权限踩坑预警 (`not writable by user`)**  
> 若挂载目录报无写入权限错误，**绝对禁止在宿主机用 `chown` 强行篡改 `root` 所有权**！该故障通常由于宿主机遗漏了附加组群或未开启目录 SGID 遗传导致。详尽无痛排障指令请见：[5.2 Servarr 无法写入 Media 目录](../../../docs/06-常见问题与故障排除.md#52--servarr-无法写入-media-目录-not-writable-by-user)。

### 4.2 Prowlarr 索引器

自动化脚本 `startup.sh` 已完成常用公共索引器的基础注入。
若需补全私有索引器资源，请访问 Prowlarr Web UI (`http://YOUR_IP:9696`)，在 **Indexers** 页面手动添加您的私有站点。

> [!TIP]
> **批量运维技巧：防范“下载元数据”卡死**  
> 为了避免抓取到大量无 Peer 的磁力链接，建议利用 Prowlarr 的 **Bulk Edit** 功能，开启 **Show Advanced** 后将所有公网站点的 **Minimum Seeders** 统一设置为 `5`。  
> 详细操作指引详见：[8.1 磁力链接卡在“下载元数据”故障排除](../../../docs/06-常见问题与故障排除.md#81--磁力链接卡在“下载元数据”-downloading-metadata-或“失速”-stalled)。

### 4.3 Bazarr 字幕源配置

由于部分高价值字幕站需要独立账密，访问 Bazarr Web UI (`http://YOUR_IP:6767`)，通过 **Settings → Providers** 进行挂载：

1. **基础外挂**：添加 **OpenSubtitles**（需提前在官网注册账号）。
2. **动漫进阶**：为 **Anime Tosho** 提供商配置 AniDB API Client，以显著提升番剧的字幕命中率。

## 五、 高级配置与扩展功能

### 5.1 搜索逻辑与自动化配置

Sonarr/Radarr 的设计偏向"监控新发布"，而非"搜索旧资源"。如果添加剧集后没有立即下载：

1. **添加时设置**：在添加剧集时，勾选 **"Start search for missing episodes"**
2. **剧集类型**：对于动漫，将 `Series Type` 改为 **Anime**，提升精准匹配率
3. **手动补课**：进入剧集详情页，点击"放大镜"图标触发全局搜索
4. **交互式搜索**：点击单集旁边的"人像"图标，查看候选资源及排除原因

### 5.2 Recyclarr 质量配置自动同步

Recyclarr 每天凌晨 3:00 自动同步 [TRaSH Guide](https://trash-guides.info/) 的最佳质量配置到 Sonarr/Radarr，包括：

- 质量定义 (Quality Definitions)
- 质量配置文件 (Quality Profiles)
- 自定义格式 (Custom Formats) 及其评分

配置模板位于 `setup/recyclarr.yml.template`，首次启动时由 `startup.sh` 自动注入 API Key 并生成最终配置。

### 5.3 中文字幕 Custom Formats

`startup.sh` 会自动导入以下中文字幕识别格式到 Sonarr/Radarr/Whisparr：

| CF 名称 | 匹配规则 | Profile 分数 |
| :--- | :--- | :--- |
| Language: Chinese Simplified (CHS) | CHS、简体、简中、GB、简日、SC | 100 |
| Language: Chinese Traditional (CHT) | CHT、繁体、繁中、BIG5、繁日、TC | 100 |
| Language: Chinese Subtitle | 中字、中文字幕、中文、双语、中英 | 50 |
| Hardcoded Chinese Subtitle | HC、hardcode、hardsub、内嵌、嵌字 | — |

## 六、 日常运维

- **Recyclarr 手动同步**：`docker exec recyclarr recyclarr sync`
- **qBittorrent 忘记密码**：删除 `config/qbittorrent/qBittorrent.conf` 中的 `WebUI\Password` 行并重启容器，将生成新的临时密码
- **查看日志**：`docker logs <容器名>` 查看各服务运行日志
