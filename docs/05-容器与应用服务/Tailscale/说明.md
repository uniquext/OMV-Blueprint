# Tailscale - 极速安全的虚拟内网 (WireGuard)

> Tailscale 是基于 WireGuard 的零配置 VPN 组网工具，无需公网 IP 即可构建安全的点对点虚拟内网，支持子网路由和 Exit Node。
>
> - **官方文档**：[Tailscale Docs](https://tailscale.com/kb)
> - **项目地址**：[tailscale/tailscale](https://github.com/tailscale/tailscale)

---

## 一、 部署架构 (Deployment)

Tailscale 采用单容器 Host 网络模式部署，需要访问宿主机 TUN 设备以建立 WireGuard 隧道，实现 P2P 直连或 DERP 中继组网。

### 1.1 容器架构说明

- **tailscale**：Tailscale 客户端守护进程，负责与协调服务器通信、建立 WireGuard 隧道、维护节点间连接状态

### 1.2 路径映射逻辑

- **Service Local (服务自产层)**：
  - `./state` 目录：Tailscale 运行状态与认证数据持久化存储，包含节点密钥和连接状态
- **AppData (公共资源层)**：无
- **Resource (媒体资源层)**：无

## 二、 快速启动 (Quick Start)

> [!IMPORTANT]
> **前置检查**：部署前，请确保已在 Tailscale 管理控制台生成 Auth Key，并填入 `private.env` 中的 `TAILSCALE_AUTHKEY`。

### 前置步骤

#### 1. 生成认证密钥 (Tailscale 管理控制台)

1. 访问 [Tailscale Keys](https://login.tailscale.com/admin/settings/keys)
2. 点击 **Generate auth key**
3. 推荐配置：
   - **Reusable**：✅ 勾选（适合长期运行的服务器，避免密钥过期后需重新部署）
   - **Pre-approved**：✅ 勾选（跳过管理员审批流程，节点自动加入网络）
4. 复制生成的 Auth Key

#### 2. 填写私有配置

1. 编辑 `AppData/inject_env/private.env`
2. 将 `TAILSCALE_AUTHKEY` 替换为上一步获取的密钥
3. 执行注入脚本：`sudo bash inject.sh private.env`

### 部署步骤

1. 查阅与验证配置逻辑：
   - 配置文件：[Tailscale.yml](../../../Compose/Tailscale/Tailscale.yml)
2. 在 OMV Web UI 的 Compose 标签页，加载此 `Tailscale.yml` 文件。
3. 点击 `Up`（启动）即可。
4. 验证节点状态：
   ```bash
   # 查看容器日志，确认认证成功
   docker logs tailscale

   # 在容器内检查 Tailscale 状态
   docker exec tailscale tailscale status
   ```

## 三、 详细配置参数表 (Configuration Table)

所有环境变量声明于 `Tailscale.yml`，且严格依赖宿主机的 `global.env` 与 `private.env` 透传。

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| **基础配置** | | | |
| 容器映像 | `tailscale/tailscale:latest` | 🟢 保持默认 | Tailscale 官方镜像 |
| 网络模式 | `host` | 🟢 保持默认 | Host 模式确保 WireGuard 隧道可正常建立 P2P 连接 |
| **环境变量 (Environment)** | | | |
| 权限 UID | `PUID` | 🟢 保持默认 | 指向 `${PUID}`（OMV `appuser` 的数值 1001） |
| 权限 GID | `PGID` | 🟢 保持默认 | 指向 `${PGID}`（`users` 组的数值 100） |
| 时区同步 | `TZ` | 🟢 保持默认 | 指向 `${TZ}`（如 `Asia/Shanghai`），防止时间戳错位 |
| 认证密钥 | `TS_AUTHKEY` | 🔴 必须对应 | 指向 `${TAILSCALE_AUTHKEY}`，从管理控制台获取的 Auth Key |
| 状态目录 | `TS_STATE_DIR` | 🟢 保持默认 | `/var/lib/tailscale`，Tailscale 持久化状态存储路径 |
| 用户空间网络 | `TS_USERSPACE` | 🟢 保持默认 | `false`，禁用用户空间网络栈以使用内核 WireGuard |
| **目录挂载 (Volumes)** | | | |
| 主机时间挂载 | `${LOCALTIME_PATH}` | 🟢 保持默认 | `/etc/localtime` 确保日志的时间属性严格一致 |
| 状态数据目录 | `./state` | 🟢 保持默认 | Tailscale 认证状态与节点密钥持久化存储 |
| **设备映射 (Devices)** | | | |
| TUN 设备 | `/dev/net/tun` | 🟢 保持默认 | WireGuard 隧道必需的虚拟网络设备 |
| **能力添加 (Capabilities)** | | | |
| 网络管理 | `net_admin` | 🟢 保持默认 | 允许容器配置网络接口和路由表 |
| 原始套接字 | `net_raw` | 🟢 保持默认 | 允许容器发送原始网络数据包 |

## 四、 首次登录配置 (Initial Setup)

容器启动后，Tailscale 会使用 Auth Key 自动完成认证并加入网络。首次部署后建议进行以下配置：

### 4.1 验证节点状态

在 [Tailscale 管理控制台](https://login.tailscale.com/admin/machines) 确认节点已成功加入网络。

### 4.2 配置子网路由（可选）

如需通过 Tailscale 节点访问 OMV 所在局域网的其他设备，需开启子网路由：

1. **容器端广播子网**：
   ```bash
   docker exec tailscale tailscale up --advertise-routes=192.168.1.0/24
   ```
   将 `192.168.1.0/24` 替换为您的实际局域网网段

2. **管理控制台审批**：在 Machines 页面点击节点右侧的 `...` > **Edit route settings** > 勾选子网路由并审批

### 4.3 配置 Exit Node（可选）

如需将 Tailscale 节点作为出口节点（即其他设备通过此节点上网）：

1. **容器端广播 Exit Node**：
   ```bash
   docker exec tailscale tailscale up --advertise-exit-node
   ```

2. **管理控制台审批**：在 Machines 页面审批 Exit Node

## 五、 高级配置与扩展功能

### 5.1 Tailscale SSH

Tailscale SSH 允许通过 Tailscale 网络直接 SSH 到节点，无需密钥或密码：

1. **启用 Tailscale SSH**：
   ```bash
   docker exec tailscale tailscale up --ssh
   ```

2. **在管理控制台配置 ACL**：在 Access Controls 中添加 SSH 策略

### 5.2 Taildrop 文件传输

Tailscale 网络内的节点间可直接使用 Taildrop 传输文件：

```bash
# 从其他 Tailscale 节点发送文件到此 OMV 节点
tailscale file cp myfile.txt omv-nas:

# 在容器内查看接收的文件
docker exec tailscale tailscale file get /tmp
```

### 5.3 MagicDNS

Tailscale 自动为每个节点分配易读的域名（如 `omv-nas.tailnet-name.ts.net`），无需记忆 IP 地址。此功能默认开启。

## 六、 日常运维

- **查看节点状态**：`docker exec tailscale tailscale status`
- **查看节点 IP**：`docker exec tailscale tailscale ip`
- **重新认证**：`docker exec tailscale tailscale login`
- **断开连接**：`docker exec tailscale tailscale down`
- **查看日志**：`docker logs tailscale`
- **更新密钥**：如 Auth Key 过期，更新 `private.env` 中的 `TAILSCALE_AUTHKEY` 并重启容器
