# Cloudflared - 外网安全访问隧道

> Cloudflare Tunnel 客户端，无需公网 IP 即可安全暴露内网服务到公网。
>
> - **官方文档**：[Cloudflare Tunnel Docs](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/)

---

## 一、 部署架构 (Deployment)

Cloudflared 采用单容器 Host 网络模式部署，通过在本地运行一个轻量级容器，与 Cloudflare 边缘节点建立加密隧道，从而将内网服务安全地暴露到公网，**完全不需要公网 IP，也不需要在路由器上做任何端口转发**。

### 1.1 容器架构说明

**单容器服务**：
- **cloudflared**：Cloudflare Tunnel 客户端，负责建立和维护与 Cloudflare 边缘服务器的加密隧道连接。

### 1.2 路径映射逻辑

- **Service Local (服务自产层)**：
  - `./config` 目录：Cloudflared 配置文件存储位置，包含隧道证书和配置信息。

## 二、 快速启动 (Quick Start)

> [!IMPORTANT]
> **前置检查**：部署前，请确保已在 Cloudflare Zero Trust Dashboard 创建 Tunnel 并获取 Token。

### 前置步骤

#### 1. 准备工作 (Cloudflare 网站)

1. **添加站点**：在 Cloudflare 仪表板添加您的域名，选择 **Free** 计划。
2. **域名 DNS 迁移 (关键)**：将域名 DNS 服务器修改为 Cloudflare 提供的地址（如 `dash.ns.cloudflare.com`）。
3. **配置 DNSSEC (精细化指引)**：
   - 在 Cloudflare 开启 DNSSEC 后获取参数。
   - **在域名商后台**（如阿里云）添加 DS 记录：
     - **算法 (Algorithm)**: 请选择 **`ECDSA Curve P-256 with SHA-256`** (代码 **13**)。
     - **摘要类型 (Digest Type)**: 选择 **`SHA-256`** (代码 **2**)。

#### 2. 创建隧道 (Zero Trust 控制台)

1. **进入路径**：**Zero Trust** > **Networks** > **Tunnels**。
2. **创建 Tunnel**：点击 **Create a Tunnel**，命名（如 `omv-nas`），选择 **Docker** 环境并复制 **Token**。

#### 3. OMV 端部署与核心参数 (Docker)

1. 编辑 `Cloudflared.env` 文件
2. 将 `CF_TUNNEL_TOKEN` 替换为从 Dashboard 获取的实际 Token

### 部署步骤

1. 查阅与验证配置逻辑：
   - 配置文件：[Cloudflared.yml](../../../Compose/Cloudflared/Cloudflared.yml)
   - 环境变量：[Cloudflared.env](../../../Compose/Cloudflared/Cloudflared.env)
2. 在 OMV Web UI 的 Compose 标签页，加载此 `Cloudflared.yml` 文件。
3. 点击 `Up`（启动）即可。

## 三、 详细配置参数表 (Configuration Table)

所有环境变量声明于 `Cloudflared.yml`，且严格依赖宿主机的 `global.env` 透传。

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| **基础配置** | | | |
| 容器映像 | `cloudflare/cloudflared:latest` | 🟢 保持默认 | Cloudflare 官方 Tunnel 客户端镜像 |
| 网络模式 | `host` | 🟢 保持默认 | Host 模式允许隧道访问宿主机所有网络接口 |
| 运行命令 | `tunnel run --token ${CF_TUNNEL_TOKEN}` | 🟢 保持默认 | 使用 Token 模式启动隧道 |
| **环境变量 (Environment)** | | | |
| 权限 UID | `PUID` | 🟢 保持默认 | 指向 `${PUID}`（OMV `appuser` 的数值 1001） |
| 权限 GID | `PGID` | 🟢 保持默认 | 指向 `${PGID}`（`users` 组的数值 100） |
| 时区同步 | `TZ` | 🟢 保持默认 | 指向 `${TZ}`（如 `Asia/Shanghai`），防止时间戳错位 |
| Tunnel Token | `CF_TUNNEL_TOKEN` | 🔴 必须对应 | 从 Cloudflare Dashboard 获取的隧道认证令牌 |
| **目录挂载 (Volumes)** | | | |
| 主机时间挂载 | `${LOCALTIME_PATH}` | 🟢 保持默认 | `/etc/localtime` 确保日志的时间属性严格一致 |
| 配置文件目录 | `./config` | 🟢 保持默认 | 存储隧道证书和配置文件 |

## 四、 首次登录配置 (Initial Setup)

部署成功后，需在 Cloudflare Zero Trust Dashboard 配置公共主机名：

- **添加路由**：在隧道详情中点击 **+ Add route** > **Published application**。
- **映射参数**：
  - **Subdomain**: `nas` (对应访问地址 `nas.[YOUR_DOMAIN]`)
  - **Domain**: `[YOUR_DOMAIN]`
  - **Service URL**: `http://localhost:80` (指向 OMV 管理界面)

同理可以添加其他服务，例如：
- `[service].[YOUR_DOMAIN]` -> `http://localhost:port`

## 五、 高级配置与扩展功能

### Access 访问保护

Cloudflare Access 可以为您的内网服务提供企业级的身份验证保护。

#### 网页 MFA 访问保护 (OTP)

每次访问受保护域名时，都需要输入邮箱接收验证码：

1. **添加 IDP**：**Settings** > **Authentication** > **Identity providers** > **One-time PIN**。
2. **创建 Policy**：
   - **Action**: `Allow`
   - **Include**: 选择 `Emails` 并填入您的授权邮箱。
3. **创建 Application**：
   - 类型选择 **Self-hosted**。
   - **Domain**: 填入您的访问地址（如 `nas.domain.com`）。
   - **Access policies**: 勾选上一步创建的策略。

#### 客户端 APP 保护 (Service Token)

适用于如 Immich 等无法处理网页跳转登录的客户端：

1. **创建 Token**：**Access** > **Service Auth** > **Service Tokens** > **Create Service Token**。
   - **务必保存**：`Client ID` 和 `Client Secret`（仅显示一次）。
2. **配置 Policy**：
   - **Action**: `Service Auth`
   - **Include**: 选择 `Service Token`。
3. **客户端接入**：
   - 在客户端 APP 连接地址中使用受保护域名。
   - 配置自定义 Headers：`Cf-Access-Client-Id` 和 `Cf-Access-Client-Secret`。
