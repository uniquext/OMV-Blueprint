# Clash Meta (mihomo) - 代理网关

> 高性能代理网关，提供可视化的代理节点管理和规则切换功能。

---

## 一、 部署架构 (Deployment)

Clash Meta 采用双容器桥接网络部署，已移除对宿主机网卡的透明接管层级（TUN），确保网络边界的安全收敛。只有在 `global.env` 显式引用了 `PROXY_HTTP` 的对应容器，流量才会从 Clash 端口出库。

### 1.1 容器架构说明

**多容器服务**：
- **mihomo**：代理内核服务，Clash Meta 的核心代理引擎，负责流量转发、规则匹配、订阅管理等核心功能。监听 7890 端口提供 HTTP/SOCKS5 混合代理，9090 端口提供 RESTful API 供外部调用。
- **clash-ui**：Web 管理面板，基于 metacubexd 的现代化 Web UI，提供可视化的代理节点管理、规则切换、流量统计等功能。通过浏览器访问 8080 端口即可管理代理服务。

### 1.2 路径映射逻辑

- **Service Local (服务自产层)**：
  - `./config` 目录：存放用户主配置及缓存数据库。

## 二、 快速启动 (Quick Start)

> [!IMPORTANT]
> Clash 配置强依赖预设骨架。首次启动前，请确认 `Compose/ClashMeta/config/config.yaml` 模板文件已经存在。
> 全局启用代理需要进入 `global.env` 将 `PROXY_HTTP=http://<宿主机IP>:7890` 解开注释。

1. 查阅与验证配置逻辑：
   - 配置文件：[ClashMeta.yml](../../../Compose/ClashMeta/ClashMeta.yml)
2. 在 OMV Web UI 的 Compose 标签页，加载此 `ClashMeta.yml` 文件。
3. 点击 `Up`（启动）即可。首次访问请通过浏览器打开 `IP:8080` 进入管理面板。

## 三、 详细配置参数表 (Configuration Table)

所有环境变量声明于 `ClashMeta.yml`，且严格依赖宿主机的 `global.env` 透传。

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| **基础配置** | | | |
| 容器映像 | `metacubex/mihomo:latest` | 🟢 保持默认 | Clash Meta 核心代理引擎镜像 |
| 网络模式 | `default` (Bridge) | 🟢 保持默认 | 已彻底剥离 Host 网卡接管，改为安全的独立自定义 bridge 网络，避免冲突 |
| 代理端口 | `7890:7890` | 🟢 保持默认 | 提供给局域网其他设备或容器调用的 HTTP/SOCKS 混合总引流口 |
| API 端口 | `9090:9090` | 🟢 保持默认 | 用于给 UI 面板或脚本发送控制指令的核心 API 端口 |
| Web UI 端口 | `${CLASH_UI_PORT}:8080` | 🟡 按需调整 | 预设为 8080 端口，若宿主机该端口被冲突则修改 `ClashMeta.env` 即可 |
| **环境变量 (Environment)** | | | |
| 权限 UID | `PUID` | 🟢 保持默认 | 指向 `${PUID}`（OMV `appuser` 的数值 1001） |
| 权限 GID | `PGID` | 🟢 保持默认 | 指向 `${PGID}`（`users` 组的数值 100） |
| 时区同步 | `TZ` | 🟢 保持默认 | 指向 `${TZ}`（如 `Asia/Shanghai`），防止时间戳错位 |
| **目录挂载 (Volumes)** | | | |
| 主机时间挂载 | `${LOCALTIME_PATH}` | 🟢 保持默认 | `/etc/localtime` 确保日志的时间属性严格一致 |
| 配置目录 | `./config:/root/.config/mihomo` | 🟢 保持默认 | 用户的所有自建 config、代理提供商配置 (proxy-providers) 将落盘于此 |

## 四、 首次登录配置 (Initial Setup)

首次通过 `IP:8080` 登录 Web 管理面板后，需完成以下配置：

- **API 地址配置**
  - 在设置页面填入 API 地址：`http://<宿主机IP>:9090`
  - 点击连接测试，确保面板能正常访问代理内核
- **订阅配置**
  - 在配置页面导入您的代理订阅链接
  - 或手动编辑 `./config/config.yaml` 文件添加代理节点

## 五、 高级配置与扩展功能

### 外部协作与防灾兜底机制

**日常代理开关（一键防断网）**：
- **切勿直接在宿主机停止 Clash 容器**，这会导致已经挂载了 `http_proxy` 的其他容器完全断网。
- 日常使用如需临时关闭外网代理，请登录 UI 界面 (8080端口)，在"模式"标签页一键将 `规则(Rule)` 切换成 `直连(Direct)`。

**极端断网防跨灾应急操作（核弹兜底设计）**：
如果由于配置错误、订阅挂掉等极端意外，连代理控制面板都无法接入，并且导致所有下游容器网络瘫痪，您可以进入 `Compose/ClashMeta/` 目录：
1. **轻量抢救**：执行预埋脚本 `./emergency_bypass.sh`，脚本会将出错的 config 备份并强行用空壳白名单代替，只需2秒您的系统将瞬间直连复苏。
2. **核弹拆分（终极操作）**：执行 `./emergency_bypass.sh --nuclear`，该指令会强制擦除 `global.env` 中的代理字段并全盘重启所有堆栈。这适用于连 docker container 都无法拯救时强制剥离所有网络依赖。
