# Clash Meta (mihomo 代理网关) 说明文档

## 1. 部署架构 (Deployment)
采用**显式软路由（应用层代理）**架构。已移除对宿主机网卡的透明接管层级（TUN），确保网络边界的安全收敛边界。只有在 `global.env` 显式引用了 `PROXY_HTTP` 的对应容器，流量才会从 Clash 端口出库。

- **Service Local (服务自产层)**: `./config`，存放用户主配置及缓存数据库。
- **AppData (公共资源层)**: 本服务无需占用全局资源，纯本地化管理。
- **Resource (媒体资源层)**: 隔离。不在媒体盘产生任何碎片落盘。

## 2. 快速启动 (Quick Start)

> [!IMPORTANT]
> Clash 配置强依赖预设骨架。首次启动前，请确认 `Compose/ClashMeta/config/config.yaml` 模板文件已经存在。
> 全局启用代理需要进入 `global.env` 将 `PROXY_HTTP=http://<宿主机IP>:7890` 解开注释。

- **运行时配置**: [ClashMeta.yml](../../../Compose/ClashMeta/ClashMeta.yml)

## 3. 详细配置参数表 (Configuration Table)

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| `networks` | `default` | 🟢 建议默认 | 已彻底剥离 Host 网卡接管，改为安全的独立自定义 bridge 网络，避免冲突 |
| `PUID` / `PGID` / `TZ` | 全局透传 | 🟢 建议默认 | 基础权限与时区对齐，`mihomo` 本身虽是 root 运行，但仍需保持传参规范 |
| 代理端口 (Proxy) | `7890:7890` | 🟢 建议默认 | 提供给局域网其他设备或容器调用的 HTTP/SOCKS 混合总引流口 |
| 面板 API 端口 | `9090:9090` | 🟢 建议默认 | 用于给 UI 面板或脚本发送控制指令的核心 API 端口 |
| 面板入口 Web 端口 | `${CLASH_UI_PORT}:8080`| 🟡 根据需要 | 预设为 8080 端口，若宿主机该端口被冲突则修改 ClashMeta.env 即可 |
| `volumes` | `./config:/root/.config/mihomo`| 🟢 建议默认 | 用户的所有自建 config、代理提供商配置 (proxy-providers) 将落盘于此 |

## 4. 外部协作与防灾兜底机制 (Optimization & Fallback)

**日常代理开关（一键防断网）**：
- **切勿直接在宿主机停止 Clash 容器**，这会导致已经挂载了 `http_proxy` 的其他容器完全断网。
- 日常使用如需临时关闭外网代理，请登录 UI 界面 (8080端口)，在“模式”标签页一键将 `规则(Rule)` 切换成 `直连(Direct)`。

**极端断网防跨灾应急操作（核弹兜底设计）**：
如果由于配置错误、订阅挂掉等极端意外，连代理控制面板都无法接入，并且导致所有下游容器网络瘫痪，您可以进入 `Compose/ClashMeta/` 目录：
1. **轻量抢救**：执行预埋脚本 `./emergency_bypass.sh`，脚本会将出错的 config 备份并强行用空壳白名单代替，只需2秒您的系统将瞬间直连复苏。
2. **核弹拆分（终极操作）**：执行 `./emergency_bypass.sh --nuclear`，该指令会强制擦除 `global.env` 中的代理字段并全盘重启所有堆栈。这适用于连 docker container 都无法拯救时强制剥离所有网络依赖。
