# Jellyfin 媒体服务器

Jellyfin 是一款自由软件媒体系统，让你可以掌控媒体的管理和流媒体播放。

---

## 1. 部署架构 (Deployment)

遵照 OMV “三层架构”规范，Jellyfin 的数据结构分布如下：

- **服务自产层 (Service Local)**：映射至 `./config` 和 `./cache`，保存自身运行配置、数据库和临时缓存。
- **公共资源层 (AppData)**：字体缓存和通用脚本被映射自 `${FONT}` 和 `${COMPOSE_DATA_PATH}/scripts`，为系统级共享复用。
- **媒体资源层 (Resource)**：大容量存储影视目录位于 `${RESOURCE}/Media`。

## 2. 快速启动 (Quick Start)

> [!IMPORTANT]  
> 启动该应用前，请确保您已配置好 `global.env` 中的 `NVIDIA_GPU_TEMPLATE` 以及全局 `PUID/TZ`。如果您使用的是 Intel 核显，请参阅第 4 节打开核显直通映射。

## 3. 详细配置参数表 (Configuration Table)

所有配置参数汇总说明，基于我们的红绿灯建议机制：
- 🟢：**建议保持默认**。适用于稳定的镜像定义、网络模式等。
- 🟡：**建议根据需要调整**。适用于端口、主机名、非敏感环境变量等。
- 🔴：**必须修改**。适用于密码、账户邮箱、关键宿主机物理路径等。

| 选项 | 值/变量 | 建议 | 说明 |
| :--- | :--- | :--- | :--- |
| **容器名** | `jellyfin` | 🟢 | 统一全小写命名。 |
| **网络模式** | `bridge` (隐式) | 🟢 | 默认建立隔离网络，杜绝全端口扫描对其他容器的影响。 |
| **端　口** | `${JELLYFIN_PORT}:8096` | 🟡 | 端口由 `Jellyfin.env` 管理，默认 8096，若被占用可修改。 |
| **P/G_ID** | `${PUID}` / `${PGID}` | 🟢 | 默认由全局变量分配 (通常为 1001)，保障挂载目录权限一致性。 |
| **时　区** | `${TZ}` | 🟢 | 与宿主硬件时区挂钩，防备日志与采集错位。 |
| **GPU加速**| `extends nvidia-runtime` | 🟢 | 依赖统一硬件模块调度方案分配计算资源。 |
| **媒体包** | `${RESOURCE}/Media:/media` | 🔴 | 必须根据家庭物理存放地址在 OMV 共享中确切设置 `RESOURCE`。 |
| **代理**   | `HTTP_PROXY=${PROXY_HTTP}` | 🟡 | **默认注释**。引用 `global.env` 中的代理配置，若刮削受阻可解开注释启用代理。|
| **特定Hosts**| `extra_hosts` | 🟡 | **默认注释**。含多个 `api.themoviedb.org`，网络异常可启用直连。|

## 4. 外部协作与优化 (Optimization)

### 4.1. 硬件编解码 (NVIDIA / Intel 协同调整)
如果您**不使用** NVIDIA 显卡而是 Intel/AMD 核心显卡，可以忽略（或是保留）NVIDIA `extends` 块，**但一定要**打开 `volumes` 里面被注释掉的:
`- ${DRI}:/dev/dri`。
它会把底层的 `/dev/dri` 驱动设备直通给 Docker 进行图形解算。

### 4.2. DLNA 局域网被动发现补偿
因为抛弃了 `network_mode: "host"`，客户端设备(如电视、VR头显)通过 DLNA 或 UDP 广播在局域网扫寻服务时，可能找不到挂在 Bridge 下的服务器。
如果您有此类需求：
请在 YML 中取消 `JELLYFIN_PublishedServerUrl` 的注释，该变量已引用 `global.env` 中的 `HOST_IP` 和 `Jellyfin.env` 中的 `JELLYFIN_PORT`，会自动广播正确的访问地址。
