# Jellyfin - 媒体服务器

> 自由软件媒体系统，让你可以掌控媒体的管理和流媒体播放。
>
> - **官方文档**：[Jellyfin Docs](https://jellyfin.org/docs/)
> - **项目地址**：[jellyfin/jellyfin](https://github.com/jellyfin/jellyfin)

---

## 一、 部署架构 (Deployment)

Jellyfin 采用单容器桥接网络部署，遵照 OMV "三层架构"规范。

### 1.1 容器架构说明

**单容器服务**：
- **jellyfin**：媒体服务器，提供 Web 界面和流媒体播放功能，支持硬件加速编解码。

### 1.2 路径映射逻辑

- **Service Local (服务自产层)**：
  - `./config` 目录：保存自身运行配置、数据库。
  - `./cache` 目录：保存临时缓存。
- **AppData (公共资源层)**：
  - `${FONT}`：字体缓存，为系统级共享复用。
  - `${COMPOSE_DATA_PATH}/scripts`：通用脚本共享。
- **Resource (媒体资源层)**：
  - `${RESOURCE}/Media`：大容量存储影视目录。

## 二、 快速启动 (Quick Start)

> [!IMPORTANT]
> 启动该应用前，请确保您已配置好 `global.env` 中的 `NVIDIA_GPU_TEMPLATE` 以及全局 `PUID/TZ`。如果您使用的是 Intel 核显，请参阅第五节打开核显直通映射。

1. 查阅与验证配置逻辑：
   - 配置文件：[Jellyfin.yml](../../../Compose/Jellyfin/Jellyfin.yml)
2. 在 OMV Web UI 的 Compose 标签页，加载此 `Jellyfin.yml` 文件。
3. 点击 `Up`（启动）即可。首次访问请通过浏览器打开 `IP:8096`。

## 三、 详细配置参数表 (Configuration Table)

所有环境变量声明于 `Jellyfin.yml`，且严格依赖宿主机的 `global.env` 透传。

| 选项 | 值/变量 | 修改建议 | 说明 |
| :--- | :--- | :--- | :--- |
| **基础配置** | | | |
| 容器映像 | `jellyfin/jellyfin:latest` | 🟢 保持默认 | Jellyfin 官方镜像 |
| 容器名 | `jellyfin` | 🟢 保持默认 | 统一全小写命名 |
| 端口映射 | `${JELLYFIN_PORT}:8096` | 🟡 按需调整 | 端口由 `Jellyfin.env` 管理，默认 8096 |
| 网络模式 | `bridge` (隐式) | 🟢 保持默认 | 默认建立隔离网络，杜绝全端口扫描对其他容器的影响 |
| **环境变量 (Environment)** | | | |
| 权限 UID | `PUID` | 🟢 保持默认 | 指向 `${PUID}`（OMV `appuser` 的数值 1001） |
| 权限 GID | `PGID` | 🟢 保持默认 | 指向 `${PGID}`（`users` 组的数值 100） |
| 时区同步 | `TZ` | 🟢 保持默认 | 指向 `${TZ}`（如 `Asia/Shanghai`），防止时间戳错位 |
| GPU 加速 | `extends nvidia-runtime` | 🟢 保持默认 | 依赖统一硬件模块调度方案分配计算资源 |
| 代理 | `HTTP_PROXY=${PROXY_HTTP}` | 🟡 按需调整 | 默认注释。引用 `global.env` 中的代理配置，若刮削受阻可解开注释启用代理 |
| 特定 Hosts | `extra_hosts` | 🟡 按需调整 | 默认注释。含多个 `api.themoviedb.org`，网络异常可启用直连 |
| **目录挂载 (Volumes)** | | | |
| 主机时间挂载 | `${LOCALTIME_PATH}` | 🟢 保持默认 | `/etc/localtime` 确保日志的时间属性严格一致 |
| 配置目录 | `./config` | 🟢 保持默认 | Jellyfin 配置文件存储位置 |
| 缓存目录 | `./cache` | 🟢 保持默认 | Jellyfin 缓存目录 |
| 媒体目录 | `${RESOURCE}/Media:/media` | 🔴 必须对应 | 必须根据家庭物理存放地址在 OMV 共享中确切设置 `RESOURCE` |
| 字体缓存 | `${FONT}` | 🟢 保持默认 | 系统级字体共享 |

## 四、 首次登录配置 (Initial Setup)

首次通过 `IP:8096` 登录后，需完成以下配置：

- **语言设置**
  - 选择显示语言 → 中文（简体）
- **创建账户**
  - 设置管理员用户名和密码
- **添加媒体库**
  - 添加媒体库 → 选择类型（电影/剧集/音乐等）
  - 添加文件夹 → 选择 `/media` 下的子目录
- **元数据设置**
  - 配置元数据下载器（推荐启用 TheMovieDb）
  - 设置图片获取源

## 五、 高级配置与扩展功能

### 5.1 硬件编解码 (NVIDIA / Intel 协同调整)

如果您**不使用** NVIDIA 显卡而是 Intel/AMD 核心显卡，可以忽略（或是保留）NVIDIA `extends` 块，**但一定要**打开 `volumes` 里面被注释掉的：
`- ${DRI}:/dev/dri`。
它会把底层的 `/dev/dri` 驱动设备直通给 Docker 进行图形解算。

### 5.2 DLNA 局域网被动发现补偿

因为抛弃了 `network_mode: "host"`，客户端设备(如电视、VR头显)通过 DLNA 或 UDP 广播在局域网扫寻服务时，可能找不到挂在 Bridge 下的服务器。

如果您有此类需求：
请在 YML 中取消 `JELLYFIN_PublishedServerUrl` 的注释，该变量已引用 `global.env` 中的 `HOST_IP` 和 `Jellyfin.env` 中的 `JELLYFIN_PORT`，会自动广播正确的访问地址。

| 问题 | 现象 | 详细修复 |
| :--- | :--- | :--- |
| 字幕与封面乱码 / Skia 崩溃 | 日志报 `NullReferenceException`，字幕显示方块，拼贴图无法生成 | [→ 06-常见问题与故障排除 § 7.1](../../06-常见问题与故障排除.md#71--字幕与封面乱码-出现方块字--skia-图像编码器崩溃) |

> [!TIP]
> 此问题通常归因于容器内缺失中日韩字体及 `libfontconfig` 库。确保 `Jellyfin.yml` 中的 `${FONT}:/usr/share/fonts` 挂载生效，并在宿主机安装 `fonts-noto-cjk-extra` 即可根治。
